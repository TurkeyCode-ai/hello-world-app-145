Usage Guidelines

✅ Best Practices

  • • Use semantic HTML elements for better accessibility
  • • Follow BEM naming convention for custom components
  • • Maintain consistent spacing using the design system scale
  • • Test components across different screen sizes
  • • Ensure proper color contrast ratios (4.5:1 minimum)
  • • Use CSS custom properties for theming

❌ What to Avoid

  • • Don't use arbitrary spacing values outside the scale
  • • Avoid mixing different color palettes
  • • Don't override component styles with !important
  • • Avoid using non-semantic HTML elements unnecessarily
  • • Don't forget to test keyboard navigation
  • • Avoid inconsistent typography hierarchy
